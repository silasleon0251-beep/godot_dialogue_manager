# 基础对话

在编辑器中导航到“对话”(Dialogue)选项卡。

![对话选项卡](media/dialogue-tab.jpg)

通过点击“新建对话文件”按钮或“打开对话”按钮来打开一些对话。

![新建和打开按钮](media/new-open-buttons.jpg)

最基本的对话就是一个字符串：
这是一句对话。

如果你想添加一个正在说话的角色，那么在冒号前包含一个名字，然后是对话内容：
Nathan: 这是我正在说话。

你可以使用 [BBCode](https://docs.godotengine.org/en/stable/tutorials/ui/bbcode_in_richtextlabel.html#reference) 为你的对话增添一些趣味。除了 Godot 的 `RichTextLabel` 可用的所有功能外，你还可以使用对话管理器提供的几个额外功能：

- `[[这个|或者这个|甚至是这个]]` 用于在对话中间随机选择一个选项（注意双“[["）。
- `[wait=N]` 其中 N 是暂停对话打字的秒数。你也可以通过写入类似 `[wait="ui_accept"]` 的内容来等待操作，其中 "ui_accept" 被写成一个字符串（你*也可以*通过类似 `[wait=["ui_accept","ui_cancel"]]` 的方式等待一系列操作中的任何一个，或者仅仅为了等待任何操作，使用 `[wait]`）。
- `[speed=N]` 其中 N 是一个用于乘以默认打字速度的数字。
- `[next=N]` 其中 N 是在自动继续到下一行对话之前等待的秒数。你也可以使用 `[next=auto]` 让标签根据文本长度确定等待时间长度。

对话行是一个接一个写的：
Nathan: 我先说这个。
Nathan: 然后我会说这一行。

要为对话添加一些互动性，你可以指定回应。回应是以 `- ` 开头的行：
这是一个回应

这是一个不同的回应

这是最后一个回应

## 回应

在回应之后分支对话的一种方法是在每个回应下方嵌套更多对话。嵌套的回应对话可以无限嵌套，随着添加更多分支而扩展：
Nathan: 你开始了多少个项目但又没完成？

只有几个
Nathan: 那还不错。

很多
Nathan: 也许你应该在开始另一个之前先完成一个。

我总是完成我的项目
Nathan: 太棒了！
Nathan: ...但是完成了多少个呢？

几个
Nathan: 太棒了！

我实际上一个都没开始
Nathan: 和我想的一样。

回应可以包含决定它们是否可被选择的条件。要在回应中包含条件，添加一个包裹在方括号中的条件表达式，像这样：
这是一个普通的回应

这是一个条件回应 [if SomeGlobal.some_property == true]

## 随机化对话行

如果你想从多行中随机选择一行，你可以在行首标记一个 `%` 像这样：
Nathan: 我会说这句。
% Nathan: 然后我可能会说这句
% Nathan: 或者也许这句
% Nathan: 甚至这句？

每一行被选中的概率相等。

要设置权重，使用 `%` 后跟一个数字来表示权重。例如，一个 `%2` 意味着该行被选中的概率是普通行的两倍。
%3 Nathan: 这行有 60% 的概率被选中
%2 Nathan: 这行有 40% 的概率被选中

要分隔多组随机行，使用空行：
% 第一组
% 也是第一组

% 第二组
% 这也是第二组

你也可以让整个代码块随机：
%
Nathan: 这是第一个代码块。
Nathan: 仍然是第一个代码块。
% Nathan: 这是可能的结果。

如果第一个随机项被选中，它会完整播放两个嵌套行。

## 对话中的变量

要在对话行中显示游戏状态的某个值，将其包裹在双花括号中。
Nathan: 某个属性的值是 {{SomeGlobal.some_property}}。

同样，如果角色的名字基于变量，你也可以用双花括号提供：
{{SomeGlobal.some_character_name}}: 我的名字由玩家提供。

### 局部变量

如果你需要仅在对话期间存在的临时变量，可以使用局部变量。局部变量是仅在当前对话中存在的临时变量。当对话结束或切换对话文件时，这些变量会被删除。

*注意：`locals` 是示例气球提供的一个功能，用于演示如何处理临时状态，而不是对话管理器本身的内置功能。*

你可以通过两种方式创建局部变量：

1. **在对话中设置**，使用 `set` 或 `do`：
~start
Nathan: 你想了解什么？

说说你自己 [if not locals.asked_about_nathan]
set locals.asked_about_nathan = true
Nathan: 嗯，我是一个喜欢制作对话系统的游戏开发者。
=> start

你最喜欢什么颜色？ [if not locals.asked_favorite_color]
set locals.asked_favorite_color = true
Nathan: 我会说是蓝色。很 calming。
=> start

就这些了
Nathan: 好的，回头见！
=> END

2. **在开始对话时传递额外的游戏状态**（详情请参阅[额外游戏状态](./Conditions_Mutations.md#extra-game-states)）。来自额外游戏状态的变量可以直接引用，无需 `locals.` 前缀。

## 标签

如果你需要用标签注释你的对话行，可以将它们包裹在 `[#` 和 `]` 中，用逗号分隔。因此，要为一行的对话指定 "happy" 和 "surprised" 标签，你可以这样写：
Nathan: [#happy, #surprised] 哦，你好！

在运行时，`DialogueLine` 的 `tags` 属性将包含 `["happy", "surprised"]`。

你也可以为标签赋值，通过 `DialogueLine` 上的 `get_tag_value` 方法访问：
Nathan: [#mood=happy] 哦，你好！

对于这行对话，`tags` 数组将是 `["mood=happy"]`，而 `line.get_tag_value("mood")` 将返回 `"happy"`。

## 同步对话

如果你希望多个角色同时说话，可以使用同步行语法。在常规对话行之后，任何要与其同时说的行都可以用 `| ` 作为前缀：
Nathan: 这是一句普通的对话行。
| Coco: 我会同时说这句！
| Lilly: 我也会说这句！


要使用同步行，请访问 `DialogueLine` 上的 `concurrent_lines` 属性。

*注意：为了保持简单，内置的示例气球不包含同步行的实现。*

## 标签和跳转

标签是对话中的标记，你可以从它们开始或跳转到它们。通常，在游戏中你会通过提供一个标签来开始对话（默认标签是 `start`，但也可以是你在对话中写的任何内容）。

标签以 `~ ` 开头，并且有名称（不含空格）：
~ 这是一个标签

要从对话中的某处跳转到标签，你可以使用跳转/转向行。跳转行以 `=> ` 为前缀，然后指定要跳转到的标签：
=> 这是一个标签

当对话运行时遇到跳转时，它会将流程导向该标签标记并从中继续。

如果你想在对话中结束流程，可以跳转到 `END`：
=> END

这将结束当前的对话流程。

你也可以使用“跳转并返回”类型的跳转，它会重定向对话流程，然后返回到跳转起始点。这些行以 `=>< ` 为前缀，然后指定要跳转到的标签。一旦流程遇到 `END`（或文件结尾），流程将返回到跳转起始点并从中继续。

如果你想强制结束对话，无论是否有任何链式“跳转并返回”，你可以使用 `=> END!` 行。

跳转也可以内联用于回应：
~ start
Nathan: 那么？

第一个

另一个 => another_label

重新开始 => start
=> END

~ another_label
Nathan: 另一个？
=> END

### 表达式跳转

你可以使用表达式作为跳转指令。表达式需要解析为已知的标签名称，否则结果将不可预测。

**谨慎使用这些**，因为对话编译器无法在编译时验证表达式值是否匹配任何标签。

表达式跳转看起来像这样：

`=> {{SomeGlobal.some_property}}`

## 将对话导入其他对话

如果你有一个包含通用对话的对话文件，并希望在多个其他文件中使用，你可以将其 `import` 到这些文件中。

例如，我们可以有一个 `snippets.dialogue` 文件：
~ banter
Nathan: 等等等等。
=> END

然后我们可以将其导入另一个对话文件，并从 snippets 文件跳转到 `banter` 标签（注意 `=><` 语法，表示在跳转的对话结束后返回到这一行）：
import "res://snippets.dialogue" as snippets

~ start
Nathan: 下一行将来自 snippets 文件：
=>< snippets/banter
Nathan: 那是一些闲聊！
=> END
